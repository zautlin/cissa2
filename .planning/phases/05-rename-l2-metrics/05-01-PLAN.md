<objective>
Rename L2 metric output names from prefixed format ("L2_ASSET_EFFICIENCY") to simple format ("Asset Efficiency"), aligning with L1 and L3 naming conventions.

**Purpose:** 
Improve consistency and readability across all metric levels. Currently L1 uses "Calc MC", L3 uses "Beta", but L2 uses "L2_ASSET_EFFICIENCY" — the "L2_" prefix is redundant since metadata already tags `metric_level`.

**Output:** 
- Updated L2 service with simple metric names
- All existing and future L2 metrics use clean naming
- Consistent naming pattern across all three metric levels
</objective>

Current naming mismatch:
```
L1: "Calc MC", "ROA", "Book Equity", "Profit Margin"        ← Simple
L2: "L2_ASSET_EFFICIENCY", "L2_CAPITAL_INTENSITY", etc.     ← Prefixed
L3: "Beta", "Risk-Free Rate", "Cost of Equity"              ← Simple
```

L2 metrics defined in: `backend/app/services/l2_metrics_service.py` lines 311-362

1. L2_ASSET_EFFICIENCY → Asset Efficiency

5. L2_DIVIDEND_PAYOUT_RATIO → Dividend Payout Ratio
6. L2_ROA_BASE → ROA (same as L1, no special "base" label)

Strategy: Update the metric names in the service layer. Existing data will remain with old names, but all new L2 calculations will use new names. This is backward compatible and doesn't require data migration.

Alternative: Data migration query to rename existing L2 metrics in database (optional, can be done later if needed).
