<objective>
Automatically calculate and store all 15 L1 metrics at the completion of data ingestion, eliminating the need for manual user-triggered metric calculation.

**Purpose:** 
Improve data pipeline efficiency by pre-computing L1 metrics as part of ingestion completion. Users will have metrics ready immediately upon dataset load, reducing frontend complexity and improving perceived performance.

**Output:** 
- Refactored `MetricsService` with reusable L1 calculation logic
- Enhanced `Ingester.ingest_dataset()` that auto-triggers metrics
- Updated pipeline logger showing L1 metrics status
- All 15 L1 metrics pre-calculated and stored for every new dataset
</objective>

Key files:
- `backend/app/services/metrics_service.py` (current: 178 lines, calls SQL functions individually)
- `backend/database/etl/ingestion.py` (returns at line 199, needs hook added)
- `backend/database/schema/functions.sql` (15 functions, know their dependencies)
- `backend/database/etl/pipeline.py` (orchestrator that calls ingester, will log results)

Current flow:
1. Pipeline Stage 1b: calls `Ingester.ingest_dataset()` → returns dataset_id
2. Pipeline Stage 2: processes data
3. User (manual): calls API endpoint to calculate metrics

New flow:
1. Pipeline Stage 1b: calls `Ingester.ingest_dataset()` → calls L1 calc → returns dataset_id + metrics
2. Pipeline Stage 2: processes data
3. No user action needed (metrics pre-calculated)

Dependencies between L1 metrics (from functions.sql):
- Calc MC, Calc Assets, Book Equity: no dependencies (base metrics)
- Calc OA: depends on Calc Assets
- Calc Op Cost, Calc Non Op Cost, Calc Tax Cost, Calc XO Cost: no dependencies
- ROA, ROE, etc.: depend on base metrics being present

Strategy:
1. Extract SQL function execution logic from `MetricsService.calculate_metric()` → new method `_execute_sql_function()`
2. Create `calculate_all_l1_metrics(dataset_id)` that calls all 15 in dependency order
3. Import this into `Ingester` and call at completion of `ingest_dataset()`
4. Update ingestion return value to include L1 metrics results
5. Update pipeline logger to show L1 metrics status
</context>
