Raw Beta Range Before Rounding:  0.4 to 2.5
Typical Range:                    0.6 to 2.0
Maximum Rounding Error:           ±0.05 (±5% at typical values)

Beta Transformation Chain:
  OLS Slope (0.4-2.5) 
  → Transformed (0.6-2.0) via formula: (slope*2/3)+1/3
  → Rounded to 0.1 increment
  → Error filtered by tolerance
  → Sector averaged if Tier 1 missing
  → Floating/Fixed final beta

  Formula: KE = Rf + Beta × RP

4. MOST SENSITIVE COMPONENTS (Ranked)
================================================================================

RANK 1: Floating Beta (Cumulative Average)
  Sensitivity:   VERY HIGH
  Why:           Compounds rounding across years + step function behavior
  Observation:   Beta can get "stuck" at rounded boundary for 3+ years
  Example:       Cum avg = 0.85 → rounds to 0.9, then stuck at 0.9

RANK 2: Sector Average (Tier 2 Fallback)
  Sensitivity:   HIGH
  Why:           Averaging already-rounded values multiplies error
  Example:       Sector avg = 0.933 → rounds to 0.9 → 4% error applied to 5+ companies
  Impact:        All companies without Tier 1 data receive biased sector slope

RANK 3: Fixed Approach Beta (Ticker Average)
  Sensitivity:   MODERATE
  Why:           Single rounding step (less compounding)
  Example:       Ticker avg = 0.94 → rounds to 0.9 → 4.3% error

RANK 4: Transformation Formula
  Sensitivity:   LOW
  Why:           Mean-reversion built into formula itself
  Effect:        (slope × 2/3) + 1/3 compresses extreme values toward 1.0

5. FALLBACK TIER SENSITIVITY ANALYSIS
================================================================================

Error Tolerance Filter (Critical Threshold):
  Rule:    if rel_std_err >= tolerance: ACCEPT → Tier 1
           else: REJECT → Tier 2 fallback
  Risk:    Small std_err rounding can flip decision
  Example: rel_std_err = 0.395 (PASS) vs 0.405 (FAIL) → opposite tiers used

Sector Fallback Activation (Tier 2):
  Trigger:   When Tier 1 rejected due to high error
  Effect:    All companies in sector without Tier 1 receive sector average
  Error:     ±3% compounding effect when sector slope itself is rounded

Global Fallback (Tier 3):
  Trigger:   When 30%+ of slopes rejected across portfolio
  Effect:    Global avg = mean(surviving 70% rounded slopes)
  Risk:      Distribution skew - which rounded values survive affects global avg

10. KEY FINDINGS
================================================================================

FINDING 1: Rounding Error Compounds Through Cascade
  - Individual rounding: ±5% error
  - Sector rounding: ±3% error (from average of rounded values)
  - Floating beta rounding: Step function creates "stickiness"
  - Total effect: Moderate but measurable

FINDING 2: Sector Fallback Most Sensitive
  - When Tier 1 rejected, entire sector uses rounded sector average
  - 5-30 companies per sector compound 4% sector error
  - Effect: ±0.75% KE for Tier 2 dependent companies
  - Recommendation: Monitor Tier 2 activation carefully

FINDING 3: Floating Beta Exhibits Non-Linear Behavior
  - Cumulative average of rounded betas creates step function
  - Beta can get "stuck" at rounded boundary for multiple years
  - Example: cum_avg = 0.85 → rounds to 0.9, stays 0.9 for years 2-4
  - Expected behavior but should be communicated

FINDING 4: Large-Cap Stock Impact More Visible
  - For $100B+ companies: 2.5% Beta rounding → ~$2.5B valuation change
  - Sector-dependent (materials/industrials have higher beta variance)
  - Institutional investors sensitive to these changes

FINDING 5: Current Setting Appropriate for Intended Use
  - 0.1 rounding provides reasonable precision
  - KE error < 0.3% in typical cases
  - Better alternatives (0.01, no rounding) have greater disadvantages
  - No immediate change recommended
